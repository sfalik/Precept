## Summary

Audited every documentation surface in the repository for impact from the comma-list syntax proposal. The spike doc (`docs/working/comma-list-syntax-spike.md`) now has an exhaustive ¬ß7 coverage audit and ¬ß8 file inventory.

## Key Findings

### Grammar Generator (Definitive)

The TextMate grammar (`tmLanguage.json`) is a **build output** generated by `tools/Precept.GrammarGen/Program.cs`. It is never hand-edited.

The `fromOnHeader` structural pattern (L609‚Äì635) contains a hand-written regex (L617). Capture group 4 (state position) **already supports** comma-delimited lists. Capture group 8 (event position) accepts **only a single identifier** and must be extended.

After updating the generator, regeneration produces the corrected `tmLanguage.json`. No manual edit to the JSON file.

### Parser Disambiguation Invariant (Critical)

`docs/compiler/parser.md` L193‚Äì200 documents the peek-at-2 disambiguation invariant. Multi-state comma lists break this invariant ‚Äî `from Draft, Pending on Submit` has the `on` token at offset 4+, not offset 2. The same assumption is restated in `name-binder.md` L195‚Äì196.

This is the most critical doc update in the entire inventory. An implementer who reads the parser doc and relies on peek(2) will produce incorrect disambiguation logic.

### MCP Tools (No Code Changes)

`CatalogFormatters.FormatSyntax()` reads catalog metadata generically ‚Äî it iterates `Constructs.All` and renders slots, descriptions, and examples. No `StateTarget` or `EventTarget` string is hardcoded in any MCP tool file. When `ConstructSlot.cs` and `Constructs.cs` update, MCP output updates automatically.

### Explicitly Excluded (With Evidence)

- **`docs/language/catalog-system.md`** ‚Äî describes catalog architecture, not grammar semantics of individual slots.
- **`README.md`** ‚Äî shows valid syntax that remains correct.
- **`docs/philosophy.md`** ‚Äî no philosophical tension (¬ß5 analysis confirms).
- **`research/language/`** ‚Äî immutable evidence; never updated when features ship.
- **`docs/compiler/tooling-surface.md`** and **`docs/tooling/language-server.md`** ‚Äî describe slot-context-to-completion mappings abstractly; the mapping is unchanged.
