Preview Reimagined

Twenty-nine concepts across ten phases · Elaine · UX Exploration
Twenty-nine product shapes explored for the Precept preview panel. Concepts 21-29 are the latest MaintenanceWorkOrder layout probes, each testing a different way to keep live commands, guard feedback, and the State Diagram in view at the same time. Concept 20 (Dockable Workbench) remains the latest fully interactive dockable prototype for the same sample. Concepts 18 (Three Surfaces) + 19 (Horizontal Flow) remain the core fixed-layout comparisons. Concept 17 (Interactive Journey) is the previous interactive prototype using InsuranceClaim. Concept 14 (Contract Explorer) remains the leading static design. See deep analysis for grounded research.

⭐ Leading Direction
14
Contract Explorer

Flow Cards + Pipeline Trace + Edit-in-Place

Refines Concept 13 into a shipping-candidate shape. State cards scroll vertically with transition edges between them. The current state gets spotlight treatment (data grid, event dock, rule chips). New: (1) Inline Execution Trace — expand any event to see all 6 fire-pipeline stages with expression evaluation and guard matching. (2) Structured violation detail with source-kind badges, expression text, and target-field links. (3) Edit-in-place with toggle for editable fields. (4) What-if bar — hypothetically change a field and see all events re-evaluate. (5) All 6 outcome kinds with distinct badges (Transition, NoTransition, Rejected, ConstraintFailure, Undefined, Unmatched). (6) Mode tabs (Flow / Matrix / Notebook) in the header. Uses InsuranceClaim sample at full complexity.

⭐ recommended shipping candidate pipeline trace edit-in-place
Open →
Why this direction: Flow Cards' state-centric organization answers the developer's primary question — "I'm in state X, what can I do?" — in place. The inline pipeline trace (from Concept 12) uniquely answers "why was this event rejected?" at the expression level. The what-if bar enables exploratory debugging without committing changes. Mode tabs acknowledge that Matrix and Notebook serve different jobs (completeness review, full report) and can coexist as switchable views.

05
Notebook / Report

A Live, Scrollable Document

A vertical card-based document where each section covers one aspect: contract overview, current state, data fields (including collections with count badges), events with outcome previews, rules with structured violations. Reactive: change a field value in the data card, and all event outcome cards instantly re-evaluate. The best-scaling concept — cards scroll, sections collapse, content is linear. Works at sidebar (400px) and editor-tab (1000px) widths. Role in Contract Explorer direction: Notebook is the "full report" tab alongside Flow — best for "give me a complete view of this precept's contract" where facet-based organization wins.

Decision Matrix

Every Outcome in One Table

The entire state machine as a truth table: rows = states, columns = events, cells = outcomes. The only concept showing the complete contract at compile time — no runtime state needed. 15
Journey

Past → Present → Future as a Temporal Narrative

The preview reads top-to-bottom as a three-act story. Past: a compact trail of steps taken — each shows the event fired, the state entered, and key data deltas (field before→after). Present: the current state as hero card with full data, rules, violations, and "recently changed" markers connecting values to the step that set them. Future: available events as action cards grouped by outcome type (transitions, self-loops, blocked). Closes with an "also reachable" topology hint. Works with even 1–3 steps of history — exactly what manual simulation produces. Unlike Timeline (01), this isn't a scrubbing debugger; it's a narrative grounding the developer in their simulation path. Unlike Flow Cards (13), it organizes by time, not topology.

16
Navigator

Flow Cards with Memory — Topology + History

Takes Concept 13's whole-topology state-card layout and threads history through it. States divide into three visual layers: Visited cards glow warm with step numbers, the event that exited them, and mini data-delta strips. Current gets full spotlight with "← set at #N" provenance on changed fields. Unvisited cards dim but still show event hints of what's ahead. The left rail shows the traced path — visited nodes filled, connecting edges highlighted, step badges on each node. Transition edges between cards distinguish "traced" (already taken, warm tones) from "available" (not yet taken, cooler). The key UX question Navigator answers: "where have I been in this machine, and what's left to explore?" Could be a mode toggle on Contract Explorer (14).
