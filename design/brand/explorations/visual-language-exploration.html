Precept — Visual Language Exploration

The DSL syntax and state diagrams ARE the visual identity. The icon is one element in a broader system that includes .precept files, state diagrams, diagnostics, and the inspector panel. Background: #0c0c0f. Font: Cascadia Cove. This page now renders editor-adjacent examples with the locked semantic system: indigo for structure, one violet for all states, cyan for events, slate for data, and gold only for human-readable rule messages.

Locked decisions applied: Syntax and diagram examples on this page are aligned to the locked semantic model: one hue per category, typography for constraint pressure, and no lifecycle-tier state colors in code. Exploration here is now about composition, layout, and supporting surfaces rather than alternate syntax semantics. See brand-decisions.md and brand-spec.html for canonical references.

ParkingMeter: denser contract sample. It gets to a rule, an event ensure, comma sources, guarded transition, accumulation, dotted access, and no transition in just two states.

Decision still open: choose ParkingMeter if the hero should foreground expressive business-rule density; choose TrafficLight if the hero should foreground instant legibility and cyclic motion.

The Rubric Locked

Six criteria, each scored 1–5. Maximum 30 points. Ties broken by Precept Diff. → Emotional Hook.

Code Criterion Max What We’re Looking For
DSL Coverage DSL Coverage 5 Exercises all 9 hero requirements: rule, when, reject, dotted set, transition, no transition, 3+ states, typed event, ensure
Line Economy Line Economy 5 Fits 12–16 non-blank lines; no cramming, no padding; breathable structure with 2–3 blank separators
Domain Legibility Domain Legibility 5 Any developer — regardless of stack, industry, or seniority — understands the domain in under 3 seconds
Voice / Wit Voice / Wit 5 because messages carry domain-expert authority with optional warmth; no programmer noise, no snark
Emotional Hook Emotional Hook 5 Creates curiosity, delight, or immediate recognition on first read; the reader wants to keep reading
Precept Diff. Precept Diff. 5 Proves structural impossibility — not just state routing; at least one line that makes the product thesis tangible
Tie-breaking order: Precept Diff. → Emotional Hook. The hero must win on differentiation — that’s non-negotiable.

5. Visual identity thesis Locked

Precept's visual identity is not an icon — it is an ecosystem of four surfaces:

  1. DSL syntax — keyword-anchored lines, arrow chains, the distinctive left-edge rhythm. The shipped editor uses the locked 8-shade syntax system. This is the hero visual.
  2. State diagrams — generated directly from the precept. They use the same locked hue families, with shape rather than tint carrying origin/final structure.
  3. The icon — a mark that hints at both ideas (inscribed rules + state flow). Used at small sizes everywhere. Currently exploring.
  4. Diagnostic output — PRECEPT-prefixed codes with structured messages and the same locked category colors inline. The daily-use brand surface.

The icon does not carry the brand alone. The .precept file format and state diagrams do the heavy lifting. The semantic color system is the connective tissue: one locked authoring-time meaning per hue family, with comments in a separate editorial dusk-indigo lane and verdict colors reserved for runtime outcomes.

What is decided:
  • Primary surface = DSL syntax. Secondary = state diagrams. Tertiary = diagnostics. Icon = small mark.
  • Background: #0c0c0f everywhere. Font: Cascadia Cove (wordmark 700 small-caps, code 400-700).
  • Editor-adjacent surfaces on this page use the locked 8-shade dark-mode semantic system.
  • Comments, when shown, use italic dusk indigo #9096A6 outside the semantic palette.
  • State diagrams use the same locked hue families; origin and final status are expressed through shape, not alternate state tints.
