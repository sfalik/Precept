precept SubscriptionCancellation

state Active
state RetentionOffered
state Cancelled

field tenureMonths as number
field acceptedOffer as boolean = false

event OfferRetention
event Cancel

from Active on OfferRetention
  when tenureMonths >= 12
  transition -> RetentionOffered

from RetentionOffered on Cancel
  reject "Active retention offer must be resolved first."

Identity frame

Teach the category before you sell the tool.

The brand spec defines what Precept is, what its visual language means, and why the category reads as exact rather than theatrical.

The name is not incidental.

A precept is a strict rule of action. The product treats business constraints as precepts, so the name already contains the worldview. The spec should make that legible, not explain it away.

Why now is supporting copy.

The AI-native story matters because deterministic semantics and inspectable tooling make the contract machine-readable. It does not outrank the core category claim.

Beauty follows proof.

Precept should look exact, visible, and sound. It should never look vague, generic, or decorative-for-its-own-sake.

Canonical positioning: Precept is a domain integrity engine for .NET - a single declarative contract that binds state, data, and business rules into an engine where invalid states are structurally impossible.

Mark logic

The mark should feel governed even in silhouette.

The iconography is not a generic badge. It combines tablet form, state-machine form, and allowed flow into one governed emblem. The combined form is primary.

Tablet / precept form

The tablet establishes rule-as-object. Gold appears sparingly here because it signifies the remembered human-readable rule, not a general accent habit.

State-machine form

Equal rounded cards, centered origin dot, and emerald flow convey allowed transition. It reads as governed motion rather than process doodling.

Combined form

The tablet gives the form its law. The states and emerald arrow give it executable flow. This is the core icon logic because Precept is both rule and runtime.

Canonical semantic meaning

Every lane means one thing.

Brand owns the semantic lanes and their rationale. The system doc operationalizes them across surfaces, but it does not get to rename what they mean.

Structure#4338CA -> #6366F1

Grammar, control, and executable structure

Indigo is the structural brand anchor. It carries declaration, control, connectors, and the formal edge of the language.

  • Behavior-driving keywords are heavier.
  • Grammar words and operators stay in the same family.
  • Structure is the system's spine, not a surface accent.
States#A898F5

Where the entity is

Violet names state as condition, not mood. Shape communicates lifecycle; hue communicates that the thing is a state.

  • No lifecycle rainbow in the brand language.
  • Constraint pressure appears through italic, not extra hue.
  • Equal state shapes matter as much as color.
Events#30B8E8

What can happen

Cyan names action and change. It signals event identity without impersonating success or warning status.

  • Events stay distinct from states everywhere.
  • Runtime verdict colors never replace event meaning.
  • Italic can add rule pressure without creating a new family.
Data#B0BEC5 / #84929F

What the system knows

Bright slate carries names, types, and values. It is factual, quiet, and subordinate to structure, states, and events.

  • Names, types, and values may vary tonally.
  • The family itself does not change.
  • Data should read as information, not flair.
Rule language#FBBF24

The human-readable interruption

Gold is earned, sparse, and tied to the explanatory rule line. It is not a headline color, border habit, or general accent token.

  • Gold belongs to because and reject message content.
  • It may appear in the tablet or combined mark if it still reads as rule memory.
  • It should always feel like meaning entering the frame.

Typography intent

The code should feel like it grows out of the wordmark.

Cascadia Cove with Cascadia Code fallback is the typographic anchor. The wordmark carries defined-term authority; the code carries executable proof.

Wordmark rule

700 weight, small caps, tracked spacing

The wordmark should read like a named principle. Small caps gives it the posture of a defined term, not a startup badge.

Code rule

Same family, normal case, working weight range

Code stays in the same family so the system feels native to the editor. The brand does not jump to a different typographic voice when it starts proving itself.

Voice rule

Authoritative with warmth

The prose states facts. It does not hype. Contractions are fine. Dry wit is welcome only when the fact remains intact.

  • Lead with what Precept is.
  • Explain when clarification adds truth.
  • Never substitute mood for specificity.
Brand-side rationale

The visual system matters because the compiler already knows what each thing is.

Precept's aesthetic is grounded in semantics. The colors and typographic signals feel inevitable because they follow the structure of the DSL rather than decorating it after the fact.

Brand stance in one sentence

Precept should look like a system where the rules already exist and the interface is simply exposing them clearly.
