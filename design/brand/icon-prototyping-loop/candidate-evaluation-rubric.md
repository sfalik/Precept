Candidate evaluation rubric for Precept icon exploration.

Use this rubric before promoting any SVG into the curated top 50.

Scoring scale
- 1 = weak
- 2 = below bar
- 3 = acceptable
- 4 = strong
- 5 = exceptional

Criteria

1. Conceptual bridge
- Can you state in one sentence why this shape represents Precept? (e.g., "Docker's whale carries containers because the product carries containers." What is THIS icon's equivalent sentence?)
- Strong if the connection is specific to Precept's core ideas: rules that hold, full inspectability, impossible invalid states, one-file executable contract, prevention not detection.
- Weak if the shape is generic — it could represent any developer tool — or requires a paragraph to justify.

2. Structural soundness
- Does the icon feel like something where the rules hold? Like a mechanism that can't be in an invalid configuration?
- Weak if it looks like generic motion, a random app badge, or an arbitrary decorative shape.

3. Legibility at small size
- Does the silhouette still read at ~32px (VS Code extension icon scale)?
- Weak if meaning depends on thin lines, text labels, or too many distinct elements.

4. Distinct silhouette
- Is the overall shape memorable before you see any detail?
- Weak if it could be mistaken for another developer tool or SaaS product icon.

5. Visual economy
- Does every element earn its place? Is the composition tight and intentional rather than busy or decorative?
- Weak if it feels cluttered, over-ornamented, or designed to fill space.

6. Monochrome test
- Would the icon still work rendered in pure black and white with no color at all? Does the shape carry meaning on its own?
- Weak if the icon depends on color to communicate its concept or if the silhouette collapses without color differentiation.

7. Color and contrast
- Do the colors serve the concept and create clear visual hierarchy?
- Weak if the palette is muddy, arbitrary, or undermines legibility.

8. Uniqueness within this folder
- Is it materially different from the other candidates already kept?
- Weak if it is only a minor variant of an existing candidate.

Promotion rules
- Default promotion threshold: average score 4.0 or better.
- Hard fail: do not promote if any of Conceptual bridge, Legibility at small size, or Structural soundness is below 3.
- Tie-breaker: prefer the candidate with stronger Distinct silhouette and Visual economy.
- Near-duplicate rule: if two candidates differ only by minor routing, spacing, or stroke adjustments, keep only the stronger one.

Reference checks
- Reject anything that feels too close to a known brand from icon-reference-research.md or icon-aesthetic-references.md.
- Reject anything that reads like a screenshot fragment instead of a compact mark.

Short critique template
- Strength: one sentence.
- Risk: one sentence.
- Decision: reject, keep exploring, or promote.

Promotion note template
- Candidate: filename
- Scores: conceptual bridge / structural soundness / legibility / silhouette / visual economy / monochrome test / color and contrast / uniqueness
- Why it stays: one or two sentences
- What makes it distinct: one sentence
