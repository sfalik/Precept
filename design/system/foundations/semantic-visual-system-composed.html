Tier A / Editorial shell

The Semantic World

Orientation and framing stay markdown-led so the document remains easy to revise without flattening the composition.

A semantic system should feel authored, not merely exported.

Precept needs a document surface that reads with the same governed precision as the product itself. The point is not decoration. The point is that a reader can recognize Structure, State, Event, Data, and Rule quickly across code, runtime views, and explanation.

This composed pass is no longer just a four-band experiment. It is a fuller rehearsal for what a hybrid source system could look like when the same document carries editorial framing, reference galleries, and runtime proof without collapsing back into one giant handcrafted file.

  • Markdown stays responsible for narrative flow, section framing, and short reference notes.
  • HTML islands stay responsible for dense layouts where hand-shaped structure is the actual content.
  • A tiny local builder keeps the contract explicit and inspectable.

The reading contract still maps to the layout study:

  1. Tier A narrows and explains.
  2. Tier B widens and compares.
  3. Tier C spends width on proof.

Why the split matters

If every part of the system is forced through markdown, the visual proof becomes awkward. If every part is handwritten HTML, the document becomes expensive to revise. A hybrid source lets each section use the lighter format until the layout itself becomes the argument.

This means the document can stay legible to an AI as authored prose and structure, while still reserving authored HTML for the moments where the visual system itself is the thing being specified.

Tier A / Concept stack

Reading Model

The document defines how semantic identity, modifiers, overlays, and cross-surface translation work as a single reading grammar.

Configuration is the reading target.

The semantic visual system is trying to help a reader answer one question: what is true of this entity now? Sometimes that means field values alone. Sometimes it means field values plus lifecycle position. The document has to stay honest about both.

Four signal channels work together.

  • Colour separates construct families and verdict overlays.
  • Typography separates semantic identity from ambient explanation.
  • Form stabilizes the construct across runtime surfaces.
  • Layout determines what can be scanned, compared, or proved in a single glance.

Base depiction comes before runtime pressure.

Every construct needs a stable base depiction before runtime conditions accumulate around it. Modifiers are additive. Overlays report outcome. Neither should erase the underlying identity.

Reading claim

The document is not just describing a visual system. It is demonstrating a stable reading grammar that can survive movement between surfaces.

Translation is allowed. Drift is not.

The system can adapt to the medium. A state name in code, a state node in a diagram, and a current-state readout in runtime do not need to look identical. They do need to remain recognizably the same semantic thing.

If a document loses that continuity, it stops being a semantic visual system and becomes a style guide with nicer colors.

Tier C / Specimen split

Surface Translation

The system is only real if authored meaning survives translation into editor, diagram, form, timeline, and documentation surfaces.

Surface translation is where the semantic promise either survives or collapses.

The same authored entity can appear as code, as a state diagram, as a form, as a timeline, and as documentation chrome. The surface is allowed to adapt. The semantic identity is not.

  • Editor surfaces privilege authored law.
  • Diagram surfaces privilege topology and current position.
  • Form surfaces privilege configuration and editable pressure.
  • Timeline or runtime views privilege sequence and verdict.
  • Documentation surfaces privilege explanation without inventing a separate visual language.

Tier A / Reference prose

Documentation Conventions

The documentation layer should reinforce semantic meaning without inventing a second visual language just for explanatory pages.

Documentation should reinforce the semantic system, not step outside it.

Guide pages, reference tables, inline tokens, and explanatory callouts are still part of the product reading environment. They do not need a separate decorative language just because they are teaching instead of governing.

  • Inline construct references should keep their construct family.
  • Explanatory prose should stay in the reading voice.
  • Callouts should report status, caution, or explanation without impersonating authored constructs.
