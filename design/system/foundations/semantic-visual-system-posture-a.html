Precept

The semantic world stays legible as the surface changes.

Structure frames the contract. State positions it. Event asks what may happen. Data carries the truth being governed. Rule states the reason. The posture here is editorial and codex-like: ceremonial, readable, and visibly authored rather than productized.

Governed integrity should feel inevitable on the page, not diagrammatic by habit and not reduced to a specification table.

Hybrid semantic plate

One specimen, read as authored language and governed plate at once.

The specimen stays constant across all three studies: a compact coverage review contract with fields, a state change, an event, and one explicit reason. The art direction changes; the semantic payload does not.

Authoring surface CoverageReview.precept 
precept CoverageReview

field RequestedAmount as number default 0
field CoverageLimit as number default 0
field Rationale as string nullable

rule RequestedAmount <= CoverageLimit because "A decision cannot exceed coverage"

state Draft initial
state Reviewed

event Submit with Reason as string
on Submit ensure Reason != "" because "A reason is required"

from Draft on Submit -> set Rationale = Submit.Reason -> transition Reviewed
