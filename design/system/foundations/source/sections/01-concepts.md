### Configuration is the reading target.

The semantic visual system is trying to help a reader answer one question: what is true of this entity now? Sometimes that means field values alone. Sometimes it means field values plus lifecycle position. The document has to stay honest about both.

### Four signal channels work together.

- Colour separates construct families and verdict overlays.
- Typography separates semantic identity from ambient explanation.
- Form stabilizes the construct across runtime surfaces.
- Layout determines what can be scanned, compared, or proved in a single glance.

### Base depiction comes before runtime pressure.

Every construct needs a stable base depiction before runtime conditions accumulate around it. Modifiers are additive. Overlays report outcome. Neither should erase the underlying identity.

```text
Base depiction  ->  modifier  ->  overlay
what it is         what state     what happened
                    it is in       to it now
```

### Translation is allowed. Drift is not.

The system can adapt to the medium. A state name in code, a state node in a diagram, and a current-state readout in runtime do not need to look identical. They do need to remain recognizably the same semantic thing.

If a document loses that continuity, it stops being a semantic visual system and becomes a style guide with nicer colors.
