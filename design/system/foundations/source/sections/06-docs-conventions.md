### Documentation should reinforce the semantic system, not step outside it.

Guide pages, reference tables, inline tokens, and explanatory callouts are still part of the product reading environment. They do not need a separate decorative language just because they are teaching instead of governing.

- Inline construct references should keep their construct family.
- Explanatory prose should stay in the reading voice.
- Callouts should report status, caution, or explanation without impersonating authored constructs.

<!-- island: docs-notation -->

### The point is trust.

If the documentation surface teaches one visual rule and the runtime product shows another, the system fails exactly where a reader most needs consistency.
