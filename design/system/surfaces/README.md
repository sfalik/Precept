# Surface Specs

This folder is for canonical product-surface specs.

Each file here should describe how one surface applies the shared semantic visual system without redefining global meaning.

Examples of future surface specs:

- syntax editor
- state diagram
- timeline / preview
- inspector / data panel
- diagnostics

Surface specs may specialize layout, density, interaction, and local emphasis, but they must not redefine shared semantic families or cross-surface invariants.
