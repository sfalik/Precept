Transition Lane Refinements — K-Series Variants

Five new variants evolving from Variant K (Transition Lane), addressing Shane's feedback: no repeated current state, all information visible at rest, target states always shown with semantic modifiers (terminal, initial, current), and no hover required for critical info. Mock data: VisitorBadgePickup at ReadyForPickup.

Shane's Constraints for v4

  1. No hover required — all information (target state, blocked reason) visible at rest. Hover can enhance but never gate.
  2. Don't repeat the current state — both events originate from ReadyForPickup; showing it twice is redundant.
  3. Always show the target state — even for disabled events. The user sees where each event WOULD go.
  4. Semantic modifiers on target states — terminal (double border), initial (filled dot), current (bright solid border), unreachable (dimmed), constrained (italic).
Variant K1 Diverging Lanes K evolution

Current state shown once at the left origin, two lanes diverge rightward to their targets. The active lane is bright with event‑colour arrows; the blocked lane is muted with the reason inline. Both target states carry semantic modifier badges — PickedUp and Expired show double‑border terminal markers.

Variant K5 Transition Manifest Synthesis Recommended

The shippable version. Takes K3's vertical stack structure (narrow-panel friendly), K4's embedded verdict strips (no hover required), K1's semantic state modifiers (terminal ◎ markers), and K2's "origin lives in the header" principle (no repetition). Each transition is a self-contained row: [Event] → [Target◎] + verdict. The manifest reads as a complete decision surface — every answer the user needs is visible before any interaction.

Recommendation

Ship Variant K5 (Transition Manifest) as the transition lane pattern.

K5 is the only variant that satisfies all four of Shane's constraints simultaneously: (1) no hover required — verdicts are always visible, (2) no repeated current state — origin lives in the header badge, (3) target states always shown — even the blocked Expired state is visible at rest, (4) semantic modifiers on targets — both PickedUp and Expired carry the terminal ◎ marker with double-border treatment.

The vertical manifest structure (from K3) fits the VS Code sidebar's narrow panel width far better than K1's horizontal diverging lanes or K4's horizontal layout. The embedded verdict (from K4) makes each row self-documenting. And the "origin in the header" principle (from K2) eliminates redundancy without losing context.

Second choice: K3 (Stacked Transitions) — same vertical structure but without the verdict strips. Use K3 when vertical space is at a premium and the block reason alone is sufficient (no "Ready" confirmation needed for active events).

Best for wide panels: K1 (Diverging Lanes) — the shared-origin branching diagram is the most visually expressive, but requires ~500px of horizontal space. Reserve K1 for floating inspector windows or wide preview panels.
