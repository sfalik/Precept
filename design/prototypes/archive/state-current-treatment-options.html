Design prototype · state current treatment

Comparing current-state emphasis and unreachable-state splits.

This page extends the temporary state-treatment study without changing the canonical semantic visual system. Every option keeps the same violet family, chip anatomy, chrome, and type posture as the live artifact. The variables are only how currentness is expressed and how the two unreachable-state modifiers split apart.

Current Recommendation
Stronger fill over outline

It reads faster than the live treatment, still behaves like a state chip rather than a selected UI control, and survives stacking with initial or terminal without creating a second semantic color.

Unreachable Recommendation
Hatch + dashed split

Dim both unreachable states more aggressively than the base chip, then use a subtle interior hatch for states unreachable from the initial path and reserve the dashed perimeter for states unreachable only from the current snapshot.

Alternative Split
Dashed + dotted outline

It is cheaper to implement because both cues stay on the perimeter, but the two unreachable cases start to blur together faster than the recommended interior-versus-edge split.

Modifier study · unreachable

Unreachable-state options

The goal here is not to make unreachable states louder. It is to keep both modifiers ghosted first, then distinguish whether the disconnect is absolute from the entry path or relative to the current snapshot.

Option 05 Recommended split

Interior hatch for initial, dashed outline for current

Keep both unreachable cases dimmed first, then separate them by where the secondary cue lives. The entry-path failure takes a quiet interior hatch so the whole chip reads like a muted dead branch. The current-snapshot failure stays just as ghosted, but moves the distinction to a dashed perimeter so it reads as a from-here condition at the edge.

Strengths

The stronger dimming makes both states read unavailable before any secondary cue is parsed. The swapped mapping also tests whether the from-current case is clearer when its distinction stays on the boundary nearest the route context.

Risks

This mapping is less immediately intuitive than the original edge-for-initial split, so the dimming and hatch density have to stay disciplined. At very small sizes, the interior cue may still need a simplified fallback.

Option 06 Alternative split

Dashed outline for initial, dotted outline for current

This keeps both differences on the perimeter. It is the cheapest extension of the existing chip language because no interior treatment is required, but it also makes the two unreachable cases more similar than the recommended split.

Strengths

Low implementation cost and easy to explain. The split remains technically visible without adding texture inside the chip body.

Risks

Because both cues live on the outline, the difference weakens in dense flows, low zoom, and any context where border rhythm is already doing other work.

Evaluation frame: Precept already uses shape, weight, and typography to layer meaning. Current should therefore read as a live-state modifier, not as a second category. Unreachable cues should stay ghosted first, then differentiate whether the disconnect is authored from the entry path or only true of the present snapshot.
