1. Positioning LOCKED

Primary frame

Category creation — "domain integrity engine"

No competitor occupies this frame. Precept defines the category, then becomes synonymous with it. Same pattern as Temporal ("durable execution"), Docker ("containerization"), Terraform ("infrastructure as code").

Secondary frame

AI-native — "the contract AI can reason about"

Deterministic semantics + structured tool APIs make Precept uniquely suited for AI authoring and validation. This is the "why now" signal, not the primary identity.

Single-sentence positioning

"Precept is a domain integrity engine for .NET — a single declarative contract that binds state, data, and business rules into an engine where invalid states are structurally impossible."

2. Narrative Archetype LOCKED

Category Creator: "This is a new kind of thing."

Like Temporal's "durable execution" or Tailwind's "utility-first CSS." Requires category education but earns the strongest position.

README narrative arc

  1. One sentence: what it is (resolution)
  2. One sentence: why it's different (implicit conflict)
  3. One code sample (proof)
  4. Feature list or badge row (comprehensiveness signal)
The name "Precept" carries narrative weight — it means "a strict rule of action." The name IS the story.

4. Color System LOCKED

8 authoring-time shades in 5 families, plus 3 runtime verdict colors. Comments sit outside that semantic palette as an editorial lane. Background #0c0c0f. Every color has a fixed meaning. Nothing is decorative.

Locked design

Precept syntax highlighting is a dark-mode-only 8-shade system with no user palette overrides. One hue identifies each category; italic signals constraint pressure. States, events, and data names no longer use extra shades to express lifecycle or rule presence. Gold is reserved for human-readable rule messages.

Semantic color as brand

Most tools pick colors for aesthetics, then assign meanings later. Precept's palette is the reverse — every color exists because the compiler knows something about the code and needs a way to say it. The system is organized along two dimensions:

  • Dimension 1 — category: Structure, States, Events, and Data each get a dedicated visual lane so a line is scannable before it is read.
  • Dimension 2 — constraint pressure: Italic means the token is constrained or guarded. Same hue, different posture.
  • Rules are mostly absorbed into structure — assert, reject, when, and no are structural drivers. Gold is reserved for human-language messages.
  • Verdicts stay runtime-only — enabled / blocked / warning remain outside authoring-time syntax, so green and red cannot be mistaken for static code meaning.

Structure · Indigo · 239–245°

Two indigo shades split DSL control from DSL grammar. Bold marks the words that drive behavior; normal weight carries the connective tissue.

Semantic #4338CA
bold · precept, field, state, event, invariant, from, on, in, set, transition, assert, reject, when
Grammar #6366F1
normal · as, with, default, nullable, any, of, into, because, operators, punctuation, ->, =

States · Violet · 260°

Nouns — where you are. One violet identifies all states. Italic means the state participates in assert-based constraints.

State #A898F5

Events · Cyan · 195°

Verbs — what happens. One cyan identifies events. Italic means the event has assert-based validation.

Event #30B8E8
one hue for all events · normal by default, italic when constrained

Data · Bright Slate · 215°

Variables — what you know. Data splits into names, types, and values. Only guarded names use italic.

Names #B0BEC5
normal / italic if guarded · field names, arg names
Types #9AA8B5
normal · string, number, boolean, set, queue, stack
Values #84929F

Rules · Gold · 45°

Gold is reserved for human-readable rule messages. The message is the visual interrupt; the rule keywords themselves stay in structure.

Messages #FBBF24
normal · because / reject string content

Comments · Editorial Outside The Palette

Comments are visible but intentionally excluded from the executable semantic lanes. They use dusk indigo so they read as annotation rather than state, event, data, or rule content.

Comment #9096A6
italic · # comments only, outside the 8 semantic shades

Verdicts · Runtime only

Never in syntax highlighting — reserved entirely for runtime outcomes. Emerald is purely "enabled" (no longer double-duty with types). Green/red/yellow can't be confused with authoring-time colors because they only appear in the inspector and diagrams.

6. Visual Language LOCKED

Primary surface — DSL syntax as hero

The .precept file format IS the brand. The code is the hero image.

precept LoanApplication field ApplicantName as string nullable field RequestedAmount as number default 0 invariant RequestedAmount >= 0 because "Amount cannot be negative" state Draft initial state UnderReview state Approved state Declined event Submit with Applicant as string on Submit assert Applicant != "" because "Name required" from Draft on Submit -> set ApplicantName = Submit.Applicant -> transition UnderReview in Approved assert ApplicantName != null because "Approved loans must have an applicant" from UnderReview on Approve -> transition Approved from UnderReview on Decline -> transition Declined

Secondary surface — state diagrams

The VS Code preview panel uses the same hue families. Shape, not tint, carries initial/final structure.
