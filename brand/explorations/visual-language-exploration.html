1. The .precept file IS the hero visual

Like Prisma's .prisma schema or Tailwind's utility classes, the DSL syntax is the most frequently seen brand surface. Every keyword-anchored line creates a distinctive left-edge pattern.

loan-application.precept — as hero code block

Amber keywords on dark background. The left-edge keyword rhythm is the visual signature.

precept LoanApplication # Fields define the data the workflow carries. field ApplicantName as string nullable field RequestedAmount as number default 0 field CreditScore as number default 0 field DocumentsVerified as boolean default false # Invariants are always-true constraints. invariant RequestedAmount >= 0 because "Requested amount cannot be negative" # States define the workflow phases. state Draft initial state UnderReview state Approved state Declined # State constraints. in Approved assert DocumentsVerified because "Approved loans must have verified documents" # Events with argument constraints. event Submit with Applicant as string, Amount as number on Submit assert Applicant != "" because "Applicant is required" # Transitions: the workflow's behavior, end to end. from Draft on Submit -> set ApplicantName = Submit.Applicant -> set RequestedAmount = Submit.Amount -> transition UnderReview
The visual identity move: In the README, landing page, and docs, the .precept code block is the hero — not a decorative illustration. The syntax highlighting palette, keyword rhythm, and arrow chains are what people remember. This is the Tailwind/Prisma approach.

5. Visual identity thesis

Precept's visual identity is not an icon — it's an ecosystem of four surfaces:

  1. DSL syntax — keyword-anchored lines, arrow chains, the distinctive left-edge rhythm. This is the hero visual.
  2. State diagrams — nodes and labeled edges, generated directly from the precept. The companion graphic.
  3. The icon — a mark that hints at both ideas (inscribed rules + state flow). Used at small sizes everywhere.
  4. Diagnostic output — PRECEPT-prefixed codes with structured messages. The daily-use brand surface.

The icon doesn't have to carry the entire brand. It's one element in a system where the .precept file format and state diagrams do the heavy lifting.
