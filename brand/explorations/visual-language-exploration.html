Background: #0c0c0f. Font: Cascadia Cove. This page now renders editor-adjacent examples with the locked semantic system: indigo for structure, one violet for all states, cyan for events, slate for data, and gold only for human-readable rule messages.

Locked decisions applied: Syntax and diagram examples on this page are aligned to the locked semantic model: one hue per category, typography for constraint pressure, and no lifecycle-tier state colors in code. 1. The .precept file IS the hero visual Locked

Like Prisma's .prisma schema or Tailwind's utility classes, the DSL syntax is the most frequently seen brand surface. Every keyword-anchored line creates a distinctive left-edge pattern. In the locked system, color means category, not lifecycle: the code IS the hero image.

loan-application.precept

Four sections, one specification. The data it carries, the phases it moves through, what can happen, and how it behaves. You are not writing code — you are describing how the workflow should work.

precept LoanApplication field ApplicantName as string nullable field RequestedAmount as number default 0 field CreditScore as number default 0 field DocumentsVerified as boolean default false invariant RequestedAmount >= 0 because "Requested amount cannot be negative" state Draft initial state UnderReview state Approved state Funded state Declined in Approved assert DocumentsVerified because "Approved loans must have verified documents" event Submit with Applicant as string, Amount as number, Score as number on Submit assert Applicant != "" because "An applicant name is required" on Submit assert Amount > 0 because "Loan requests must be positive" event Approve with Amount as number event Decline with Note as string event VerifyDocuments event FundLoan from Draft on Submit -> set ApplicantName = Submit.Applicant -> set RequestedAmount = Submit.Amount -> set CreditScore = Submit.Score -> transition UnderReview from UnderReview on VerifyDocuments -> set DocumentsVerified = true -> no transition from UnderReview on Approve when DocumentsVerified && CreditScore >= 680 -> transition Approved from UnderReview on Approve -> reject "Approval requires verified documents and strong credit" from UnderReview on Decline -> transition Declined from Approved on FundLoan -> transition Funded
The authoring experience is the value proposition. A precept reads top to bottom in natural sections — data, states, events, behavior — each one self-explanatory. Whether you write it by hand or an AI generates it from a conversation, the result is immediately readable. The coloring reinforces the sections: you can scan the left edge and know exactly where you are.

Shortlisted — tightened hero candidates Exploring

Two candidates at 15 lines each — the target for a README hero block. Both compile clean and show 10 DSL features each, but they emphasize different strengths.

B′) ParkingMeter — 15 lines
precept ParkingMeter field TimeLeft as number default 0 invariant TimeLeft >= 0 because "Negative time is a mathematical luxury we cannot afford." state Empty initial, Active event AddTime with Minutes as number on AddTime assert Minutes > 0 because "You must intend to park for at least a moment." event Inspect from Empty, Active on AddTime -> set TimeLeft = TimeLeft + AddTime.Minutes -> transition Active from Active on Inspect when TimeLeft == 0 -> transition Empty from Active on Inspect -> no transition
10 features: precept, field+default, invariant+because, state+initial, event with, assert+because, comma sources, when guard, set+dotted access, no transition.
H′) TrafficLight — 15 lines
precept TrafficLight field QueuedCars as number default 0 invariant QueuedCars >= 0 because "The queue cannot owe the city cars" state Red initial, Green, Yellow event Advance with Demand as number on Advance assert Advance.Demand >= 0 because "Negative traffic would raise new questions" from Red on Advance when Advance.Demand > 0 -> set QueuedCars = QueuedCars + Advance.Demand -> transition Green from Red on Advance -> no transition from Green on Advance -> transition Yellow from Yellow on Advance -> set QueuedCars = 0 -> transition Red
10 features: precept, field+default, invariant+because, state+initial (3 states, cyclic), event with, assert+dotted access+because, when guard, no transition, set+expression, cyclic transitions.
ParkingMeter: denser contract sample. It gets to invariant, asserted event input, comma sources, guarded transition, accumulation, dotted access, and no transition in just two states.

TrafficLight: cleaner cyclic-state sample. It is faster to parse at a glance and makes the state-machine loop immediately obvious, but some of its surface area reads more like a teaching demo than a workflow contract.

Decision still open: choose ParkingMeter if the hero should foreground expressive business-rule density; choose TrafficLight if the hero should foreground instant legibility and cyclic motion.

5. Visual identity thesis Locked

Precept's visual identity is not an icon — it is an ecosystem of four surfaces:

  1. DSL syntax — keyword-anchored lines, arrow chains, the distinctive left-edge rhythm. The shipped editor uses the locked 8-shade syntax system. This is the hero visual.
  2. State diagrams — generated directly from the precept. They use the same locked hue families, with shape rather than tint carrying origin/final structure.
  3. The icon — a mark that hints at both ideas (inscribed rules + state flow). Used at small sizes everywhere. Currently exploring.
  4. Diagnostic output — PRECEPT-prefixed codes with structured messages and the same locked category colors inline. The daily-use brand surface.

The icon does not carry the brand alone. The .precept file format and state diagrams do the heavy lifting. The semantic color system is the connective tissue: one locked authoring-time meaning per hue family, with comments in a separate editorial dusk-indigo lane and verdict colors reserved for runtime outcomes.

What is decided:
  • Primary surface = DSL syntax. Secondary = state diagrams. Tertiary = diagnostics. Icon = small mark.
  • Background: #0c0c0f everywhere. Font: Cascadia Cove (wordmark 700 small-caps, code 400-700).
  • Editor-adjacent surfaces on this page use the locked 8-shade dark-mode semantic system.
  • Comments, when shown, use italic dusk indigo #9096A6 outside the semantic palette.
  • State diagrams use the same locked hue families; origin and final status are expressed through shape, not alternate state tints.
Open threads to explore:
  • Inspector panel — the runtime "what can I do from here?" view. Fifth visual surface? Uses the same palette but needs its own layout language.
  • AI/MCP as a visual surface — when an AI agent calls precept_inspect, the JSON output maps to the same categories. Could we render an "AI view" that uses the same palette?

  • Icon direction — autonomous loop running. Will incorporate locked palette colors once a form converges.
